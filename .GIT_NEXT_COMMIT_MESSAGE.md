fix: portal context menu to document.body to escape transform/overflow clipping (BUG19)

When the board is zoomed, CSS `transform: scale()` on .kanban-board creates
a containing block that breaks `position: fixed` on the context menu.
Combined with `overflow: hidden` on .zoom-wrapper, the menu is completely
clipped and invisible.

Fix: add a Svelte `use:portal` action that appends the context menu to
document.body, escaping all transform/overflow ancestors.
